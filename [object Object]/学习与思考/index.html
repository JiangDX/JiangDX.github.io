学习与思考

发表于 2018-08-23   |   更新于 2018-08-24 | 分类于 个人日志 | 本文总阅读量 次
字数统计: | 阅读时长 ≈

我偶尔会遇到学习状态比较好的时候，但是相当一部分时间，我的学习效率是相当低下的，这让我时常误以为是自己天赋太糟。仔细想想，其实是只顾着奔波忙碌，忘了停下来思考。

有这样一则故事，著名物理学家卢瑟福（原子核模型就是他提出来的）一天深夜走进实验室，发现一名学生还在做实验，卢瑟福问他，为什么这么晚还不睡，学生答道，我在做实验，卢瑟福问，那么白天你在做什么，学生说，也是在做实验，卢瑟福又问：那么你用什么时间思考呢？

孔夫子说过，学而不思则罔，如果没有思考，所谓的十年工作经验，也不过是一年的经验重复了十次而已。

学习

输入——整合——回顾——输出

  • 输入：视频、书籍、讲课、记录

    记录不能视为笔记，两者应该是区分开的，记录可以细分，然后整合成笔记、索引、补充以及一般记录。

    另外现在的教材大多晦涩难懂，我建议看过之后把它用自己的方式再写一遍，否则和没看没什么区别。（我的方式就是下面会说到的“是什么——怎么做——为什么”）

  • 整合：笔记、索引、记忆

    • 笔记记录学科思想、底层方法以及核心操作。
    • 对于技术操作，可以建立索引，因为记忆是会丢失的，所以与其耗费大力气去记笔记和记忆，不如建立索引。就比如一开始讲的卢瑟福的故事，我并没有把它记下来，只是知道有这么个故事，然后查了一下我的索引。
    • 笔记和索引离不开工具，我一直提倡用纸质笔记本，不过我也承认数字版也相当有好处。我突然觉得这一部分很重要，所以我想展开讲。
      • 我的纸质笔记系统是由便利贴构建的，我买了四色的便利贴，每种颜色用于不同的用途，比如黄色是我写笔记的，绿色用来写观后感什么的，有时候也会清理出一块不贴便利贴的区域，通常是用来做复习用的。
      • 我的索引系统交给了Pocket + SimpRead插件，我非常喜欢它们，甚至我都不想解释原因。Pocket是一个跨平台的稍后阅读的平台，而且我几乎可以把所有想保存的文章、图片放到Pocket，而且没有什么限制。Pocket没有分类，只有标签，但这也是它的好处，它能让我不再纠结分类，而且把索引做得更好。而SimpRead插件支持Chrome和Firefox，可以切取内容放到Pocket，也可以从Pocket下载内容。
      • 电子笔记的话，我推荐印象笔记和Notion，这两个是不同的风格的。印象笔记是收费的，也有免费计划，当然并不很友好，但是它的强大是有目共睹的，作为笔记软件，它必须支持很多格式，甚至是在已有文件上“动刀”，显然它做到了，而且做得很好；另一个印象笔记做得很好的地方在于，印象笔记支持手写的笔记，这个功能依赖于iPad，我也是专门为了这个买的iPad，具体用法去下载“墨笔”，我不详细介绍了。另一个强大的笔记软件Notion，相对印象笔记就稍微有些不足，比如它无法去在已有的文件上写笔记，也不支持手写，Notion的强大在于高度可定制，我可以把它打造成类似于纸质笔记那样的系统；当然高度可定制是一把双刃剑，它会牺牲简洁的特性。如果想获得免费计划下相对友好的笔记软件，那么可以试一试有道云笔记（我不会告诉你们我是网易的重度用户），惊喜是有道云笔记是最早支持Markdown语法的笔记软件。
    • 记忆不止是针对笔记而言，索引也是需要记忆的。就像微信的“收藏”功能，如果没有索引与记忆的话，其实和没有收藏没什么区别，我不太相信有人会时常去随便翻翻，至少我不会。很多时候，其实我的微信收藏里有一些好东西，但是我并不记得有这么回事，事后才发现，如果早知道的话，我可以省不少力气。所以对索引的简单记忆也是必要的，而且Pocket的标签也非常适配这个事情。

  • 回顾

    建立一个记忆文件夹，时常回顾。

  • 输出：运用、分享、讨论、日志、导图

    • 运用不是指去操作已经知道的步骤，而是创造一个project，用理论基础及其推演去完成它。过程中肯定会遇到意想不到的麻烦，解决麻烦的过程也是一个学习的过程。
    • 团队讨论以及脑海风暴都是必要的，尤其是很多人会提出自己没想到的方面或是自己没踏过的坑。我曾经活跃在中国大学MOOC的讨论区上，发现里面有很多东西可以促进自己的进步，当然也不完全是这样。
    • 写日志也是一个知识输出的好办法，这个我也不多说了。或许有时候翻一翻之前的日志，会发现自己如今的进步。
    • 思维导图是表面，其实思维导图主要作用是将知识之间做一个联系和知识体系构建。我推荐手写，软件的话Mindjet和Xmind都不错，前一个收费而且不便宜，后面相对弱一些，但是有免费的解决方案。

是什么——怎么做——为什么

其实学习的步骤是根据自己的习惯联系其他方法形成的，比如说，“是什么”就是阅读中”通读“阶段，同时也是一个建立索引的阶段。

按照我自己的习惯，我是先要知道“怎么做”再去想为什么，因为“怎么做”和“是什么”是相互辅助的。这一个阶段，能够自己动手操作的一定要动手操作，动手操作比单纯的记忆更加深刻。

“为什么”就是问的一个原理，技术是死的，但是原理是要灵活推演与运用的。所以原理才是重头戏，将所有操作的原理都内化（而不只是记忆），在知识输出阶段就游刃有余。

阅读

通读——精读——整合

通读的意思是秋风扫落叶，不深究细节，除了关键部分（这个“关键部分”根据个人喜好以及知识体系需要而定）可以慢一点，圈圈画画也是很OK的。读完之后，如果是值得一读的，可以再进行精读；如果觉得这本书不咋地或者不耐看，细节或许没那么重要了。

精读就是去读细节，这个时候可以慢慢来，同时可以写一写笔记，画一个思维导图。

整合就是翻一翻笔记，写读后感之类的。做完之后可以建立一个索引。

以《三体》系列为例。通读的时候，我将第一部和后面两部分开。第一部分为两条线，一条是叶文洁的回忆，另一条是汪淼的现实经历，第二条又可以分为游戏经历和非游戏经历。第二部分就是梳理时间轴，以及每个人的人物刻画。精度的时候，就看每个人做了什么事，是怎么做的，俗话说的好，认识一个人不要听他说了什么，而是要看他做了什么事。

思考

思考这件事，也需要构建自己的系统，由于我的功底不够，所以我只能推荐两位此间高手：L先生说、道长是名思维贩子。顺便推荐一下简书，以前聚集了相当多的思考者、写手以及技术高手，可惜后来一夜没落，但是我还是保留了它。
