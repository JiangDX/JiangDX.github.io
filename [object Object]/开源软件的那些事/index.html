开源软件的那些事

发表于 2018-08-10   |   更新于 2018-08-10 | 分类于 个人日志 | 本文总阅读量 次
字数统计: | 阅读时长 ≈

原本只是想做一个小的更新，但是随着自己亲身实验开源软件，我决定专门写一篇博文，就写我试用开源软件的感想。

开源软件，成也生态 败也生态

开源最大的好处在于，如果有一个良好的生态，就会产生足够优秀的副产品（库、插件等）。但是问题在于，尤其在专业软件领域，培养开源生态并非一件容易的事情。

比如GIS领域，ArcGIS是行业龙头，我所见过的老师，就算是使用破解版，也并没有放弃ArcGIS而转向开源的GIS软件（比如我这次试用的QGIS），然后把这样的习惯又传给了学生。关于这一点，我之后还会提到。这样就造成了一个后果，没有足够的人去使用，就谈不上生态了。

另外，无论是老师还是学生，大多数都谈不上优秀的开发者（至少在国内是如此），于是生态无法建立，那么开源软件的优势就荡然无存了。

“格式”就是挡在愚公面前的太行山和王屋山

有一些产品是垄断了市场，绝大多数人都在使用它，甚至行业中使用最广泛的格式都是由它制定的，比如大家熟悉的.docx。别小看了这一个后缀名，由于MicrosoftOffice占有了绝大多数的用户，如果其他软件（比如WPS、开源的LibreOffice）不兼容.docx格式，连脚跟都站不稳，就会被清除出市场，即使再优秀也会被清除，能够完成反杀的概率不足1%。

有一个佐证，就算腾讯一直被诟病的抄袭小开发者的产品，然后利用腾讯庞大的用户流量让抄袭成为正主，直到360的出现，当然这不是这次的重点。

免费优势敌不过龙头诱惑

开源并不意味着免费，但是无论是QGIS还是FreeCAD，它们都是免费的，理论上来讲，免费的产品总能占领一席之地。但是无论是ArcGIS还是AutoCAD，找到破解版都来得太容易，所以价格优势就被削弱了。

而对于大多数人来说，开源的意识并没有那么强，所以支持开源软件并不是当务之急，面对行业龙头的诱惑，大多数人选择了后者。于是，开源软件培养生态的捷径被彻底封杀，开源软件还需要继续等待下去。
